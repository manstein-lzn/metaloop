# MetaLoop v0.1.0-alpha

MetaLoop v0.1.0-alpha is a technical preview for local single-task closed-loop agent execution.

## Included

- Local `metaloop` CLI.
- MissionSpec JSON/YAML input.
- SQLite event, checkpoint, and final-state persistence.
- Dummy worker for deterministic smoke tests.
- Codex worker backend using `codex exec --json`.
- Optional `--output-schema` with automatic fallback.
- Explicit `--no-output-schema` provider compatibility mode.
- Artifact validation for `manual`, `file_exists`, `file_contains`, `command`, and JSON parse checks.
- Multi-round Co-Design Runner with explicit `--autonomous` Codex mode for completing core MissionSpec fields from a seed intent.
- Command validators require explicit `policy.allowed_tools: ["validator.command"]`.
- `NextTaskProposal` terminal state.
- `blocked` terminal state for auth/sandbox issues.
- Alpha user guide, examples, troubleshooting, and state handoff docs.

## Not Included

- Recursive child MetaLoop spawning.
- Multi-MetaLoop Orchestrator.
- Fully polished multi-turn Co-Design UI.
- LLM-native Planner/Reviewer.
- Non-technical UI.

## Recommended First Run

```bash
source .venv/bin/activate
metaloop run "Create a dummy artifact"
metaloop run --mission examples/repo-summary.mission.json --worker codex --sandbox read-only --approval never --no-output-schema
```

For temporary workspaces outside a Git repository, add `--skip-git-repo-check`.
